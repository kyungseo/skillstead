# Social Release Note Template

**English** · [한국어](./social-release-note-template.ko.md)

Use this template to prepare a public announcement after taking a repository public.

The note does not have to live in the target repository. Keep the draft in this playbook, a private notes
repository, or a local file until it is ready to publish.

## Inputs

- Repository:
- Public URL:
- One-sentence description:
- Primary audience:
- Why it was built:
- What is ready now:
- What is intentionally incomplete:
- Screenshot, gallery, or demo asset:
- Publication channel:

## Message Checklist

- [ ] Start by saying exactly what was published.
- [ ] Explain the problem it addresses.
- [ ] Mention the most useful artifact or example.
- [ ] Avoid inflated AI claims.
- [ ] State a limitation or early-stage status honestly when relevant.
- [ ] Include the public repository link.
- [ ] Include an image or demo asset when visuals matter to the project.

## English Draft

```text
I made {project_name} public.

{one_sentence_description}

It is useful for {audience} who need to {job_to_be_done}.

The repository currently includes {key_artifact_1}, {key_artifact_2}, and {key_artifact_3}.

{limitation} is still intentionally incomplete, but the core path—{core_workflow}—is ready to use.

{repo_url}
```

## Review Questions

- [ ] Can someone unfamiliar with the project understand what it does?
- [ ] Does the announcement match the README?
- [ ] Does the image or asset show the actual project?
- [ ] Does the limitation read as honest context rather than an apology?
- [ ] Is the link public and accessible?
